실행 방법 (Hand Fist Audio Trigger)
===================================

0) 준비물
- 이 폴더(c:\ddddwoo\chapchu)에 아래 파일이 있어야 합니다.
  - index.html
  - style.css
  - script.js
  - sound1.mp3   (왼손 주먹 시 재생)
  - sound2.mp3   (오른손 주먹 시 재생)


1) 가장 권장: 로컬 서버로 실행 (HTTPS/권한 이슈 최소화)
브라우저는 보안 정책 때문에 file:// 로 열면 카메라가 막히거나,
오디오 자동 재생이 제한될 수 있습니다. 그래서 로컬 서버 실행을 권장합니다.

[방법 A] VSCode / Cursor 확장 “Live Server”
- Live Server 설치
- index.html 우클릭 → “Open with Live Server”
- 뜨는 브라우저에서 Start 버튼 클릭

[방법 B] Python이 설치돼 있으면 (PowerShell)
- 이 폴더로 이동:
  cd "c:\ddddwoo\chapchu"
- 서버 실행:
  python -m http.server 5173
- 브라우저에서 접속:
  http://localhost:5173/index.html
- Start 버튼 클릭

[방법 C] Node.js가 설치돼 있으면
- 이 폴더로 이동:
  cd "c:\ddddwoo\chapchu"
- npx로 간단 서버 실행:
  npx serve .
- 출력된 주소로 접속 후 Start 버튼 클릭


2) 브라우저에서 동작 확인 순서
1) 페이지를 열면 중앙에 Start 버튼이 보임
2) Start 클릭 → 카메라 권한 허용(Allow)
3) 화면에 손이 보이면, 손 랜드마크/선이 캔버스에 그려짐
4) 왼손을 “주먹 쥐는 순간” → sound1.mp3 1회 재생
5) 오른손을 “주먹 쥐는 순간” → sound2.mp3 1회 재생

※ 중요: 주먹을 유지하는 동안 프레임마다 반복 재생되지 않도록
  상태 플래그로 “OFF→ON 전이 순간”에만 1회 재생되게 되어 있습니다.


3) 자주 발생하는 문제 해결

(1) 카메라가 안 켜져요
- file:// 로 열지 말고 로컬 서버로 열기(위 1번)
- 브라우저 주소창 왼쪽의 카메라 권한이 “허용”인지 확인
- 다른 앱(줌/디스코드 등)이 카메라를 점유 중이면 종료

(2) 소리가 안 나요
- Start 버튼을 꼭 클릭(오디오 정책 때문에 사용자 입력이 필요)
- sound1.mp3 / sound2.mp3 경로가 맞는지 확인
  (index.html 기준 같은 폴더에 있어야 함: ./sound1.mp3, ./sound2.mp3)
- 브라우저 탭이 음소거(mute) 상태인지 확인

(3) 좌/우 손 라벨이 반대로 느껴져요
- 현재 UI는 “셀피처럼” 보이도록 좌우 반전(미러) 표시를 합니다.
- 그에 맞춰 script.js에서 모델 라벨(Left/Right)을 사용자 관점으로 보정합니다.
- 만약 미러링을 끄고 싶으면:
  - style.css에서 .video, .overlay 의 transform: scaleX(-1) 제거
  - script.js에서 MIRROR_VIEW = false 로 변경


4) 참고 (핵심 동작 요약)
- MediaPipe Hands(CDN)로 매 프레임 손 랜드마크 추출
- results.multiHandedness로 Left/Right 구분
- 손가락 tip(4,8,12,16,20) ↔ wrist(0) 거리 기반으로 주먹 여부 판정
- 주먹 “쥐는 순간”에만 오디오 1회 재생
